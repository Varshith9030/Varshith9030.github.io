Subdomain Hunter/recursive enumeration pipeline

May 2026 · Python
async enumeration passive-recon dns wildcard-detection batch-mode

Multi-phase subdomain enumeration pipeline that orchestrates 9 external tools and 25 passive API sources into one coherent run. Built for engagement contexts where coverage matters more than speed — batch-mode runs across hundreds of targets, single-target runs with maximum depth. Auto-scales resource allocation based on workload.

Pipeline phases Phase 0: external tools (subfinder, subdominator, assetfinder, amass, katana, findomain, sublist3r, bbot, puredns) · Phase 1: passive APIs (25 sources, 19 keyed + 6 free) · Phase 2: zone transfer (AXFR) · Phase 2.5: target-aware wordlist generation · Phase 3: brute force (114K-word seclists) · Phase 4: DNS record walking (CNAME, MX, NS, TXT) · Phase 5: recursive expansion (depth 4 with permutations + per-sub API recon)

Engineering highlights: per-parent wildcard DNS fingerprinting to filter false positives, dead-key tracking with masked errors, per-target output isolation for safe parallel runs, scope constraint patterns, and resume-from-file for long engagements. Outputs structured JSON with per-source attribution alongside flat text lists.

Mega-Finder/recon scanner

Nov 2025 · Python
async aiohttp recon misconfig javascript-analysis cloud-discovery

An async, high-concurrency reconnaissance engine that automates the boring parts of an external engagement. Crawl a target list, find what shouldn't be exposed, and dump structured output ready for triage. Ships with a 1M-entry wordlist and severity-classified output for direct intake into a report.

Detections .env leaks · .git repository exposure · backup archives (zip, sql, tar.gz, bak) · Swagger / OpenAPI definitions · S3 and GCS buckets · GraphQL introspection · CORS misconfigurations · directory listing · CI / CD panel exposure · JS secret extraction (API keys, tokens, Firebase configs) · phpinfo · debug endpoints

Architecture: domain-restricted HTML crawler + deep JavaScript analyzer + async HTTP probe layer with concurrency controls and rate limiting. Per-target output as JSON + CSV + the actual leaked artifacts written to disk.
